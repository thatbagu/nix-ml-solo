# Security Policy

## Reporting a Vulnerability

If you believe you've found a security vulnerability, please follow these steps:

1. **Do NOT disclose the vulnerability publicly** (no GitHub issues, public discussions, etc.)
2. Email **egor@mlship.dev** with:
   - A description of the vulnerability
   - Steps to reproduce
   - Potential impact
   - Any additional information that might be helpful

## What to Expect

- Acknowledgement within 2 business days
- Initial assessment within 5 business days
- Updates on progress as the fix is developed
- Public acknowledgement of responsible disclosure after the fix is released (unless you prefer to remain anonymous)

## Public Disclosure

Preferred process:

1. Issue reported privately
2. Fix merged to the repository
3. After an appropriate window to allow users to update, vulnerability details are published

Thank you for helping keep nix-ml-solo and its users safe.
